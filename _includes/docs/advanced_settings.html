Enable "Movies" section?

  • Yes, use a "Movies" post section (custom post type)

    • The default option. This creates a new 'Movies' custom post section type where you will create your movie posts.
  • No, use Posts but change the "Posts" menu label to "Movies"
  • No, use Posts and leave them as they are

    • In the case of Movies only, you have the option of using the default wordpress 'Posts' instead of a new custom post type. This is useful if you already have your Posts set up as the movie content area of your site, and you do not want to move/import these to the new Movie section. In this case, you want to change the setting from 'Yes' to 'No'.

    If you choose 'No' you then also have the option of renaming the default WP posts into 'Movies' or leaving them as is (‘Posts’).
  • None of the above, I only want to use Movies with shortcodes (or not at all)

    • This option disables the 'Movies' custom post type section. Choose this option if you only want to use 'Movies' via the shortcodes, or if you dont't want to use them at all.

Include mmdb css file

Default templates require the default mmdb css file. Select when to load it.
  • Yes, load it for posts that use the my-movie-database plugin only.

    • The default option. This will load it only on pages and posts where the plugin is being used.
  • Yes, but load it for all WP pages

    • Select this option only if you also want to use shortcodes outside of pages or posts, for example in archive (category) pages.
  • No

    • Selecting 'No' will never load the plugin css file.

Include bootstrap

Default templates require bootstrap libraries.Select when to load bootstrap.
  • Yes, load it for my-movie-database pages only.

    • The default option. This will load it only on pages and posts where the plugin is being used.
  • Yes, but load it for all WP pages

    • Select this option if you want to load bootstrap on all pages.
  • No

    • Selecting 'No' will never load the bootstrap libraries.

Debug Mode

This will simply output the call to TMDB used for troubleshooting. You can then click on the "see request" link that will appear at the top of the page to see the data that TMDb is sending for the requested ressource.
